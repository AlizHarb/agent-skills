# Laravel Boost Guidelines

## Purpose

Make package and application instructions useful to Laravel Boost and AI coding assistants.

## Rules

- Keep Boost guidelines action-oriented and accurate.
- Explain package installation, configuration, common APIs, and pitfalls.
- Include version-sensitive notes.
- Tell agents when to inspect installed versions before using an API.
- Avoid copying the full README into Boost files.

## Anti-Patterns

- Marketing-heavy Boost files.
- Examples that assume uninstalled packages.
- Long prose that hides operational instructions.

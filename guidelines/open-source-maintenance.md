# Open Source Maintenance Guidelines

## Purpose

Keep open-source packages trustworthy for contributors and users.

## Rules

- Provide issue templates, contribution docs, license, changelog, and security policy.
- Keep releases tagged and changelog entries dated.
- Respond to issues with reproduction steps and version details.
- Avoid breaking changes without migration notes.
- Keep CI green on the default branch.

## Anti-Patterns

- Unreleased changes piling up forever.
- Missing security contact.
- No tests for reported regressions.

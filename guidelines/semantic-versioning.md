# Semantic Versioning Guidelines

## Purpose

Choose release numbers that communicate upgrade risk clearly.

## Rules

- Use patch versions for backwards-compatible fixes.
- Use minor versions for backwards-compatible features.
- Use major versions for breaking changes.
- Document migration steps for breaking changes.
- Do not hide support drops in patch releases.

## Anti-Patterns

- Calling a breaking support drop a patch.
- Reusing an existing tag.
- Changing a tag after it has been pushed.

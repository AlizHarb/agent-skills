# Documentation Quality Guidelines

## Purpose

Make docs useful, accurate, and easy to follow.

## Rules

- Start with the user problem and fastest correct setup.
- Keep examples tested against supported versions.
- Separate quick start from advanced configuration.
- Document security, performance, and upgrade notes.
- Use consistent headings and command blocks.

## Anti-Patterns

- Huge README files with no navigation.
- Undocumented config options.
- Examples that skip required imports.

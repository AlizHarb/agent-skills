# Review Package README

## Workflow

1. Check package positioning, requirements, install command, usage examples, config, testing, and support links.
2. Verify examples match the current public API.
3. Confirm version support matches Composer and CI.
4. Ensure security, upgrade, and troubleshooting notes exist for risky features.
5. Keep README scannable with links to deeper docs.

## Verification

- README examples are accurate.
- Requirements match Composer constraints.
